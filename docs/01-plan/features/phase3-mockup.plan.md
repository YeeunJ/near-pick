## 1. 목표 (Objective)

사용자 유형별 핵심 화면 흐름(User Flow)을 텍스트·ASCII 형태로 설계한다.
각 화면에서 필요한 API를 도출하여 Phase 4(API 설계·구현)의 기반으로 삼는다.

---

## 2. 배경 (Background)

- Phase 1에서 도메인 엔티티(User, Product, Wishlist, Reservation, FlashPurchase 등)가 확립됨
- Phase 4 API 설계 전에 **어떤 화면이 있고**, **각 화면이 어떤 데이터를 필요로 하는지** 먼저 정의
- UI 구현 없이 화면 흐름만 정의 → 빠른 방향 확정

---

## 3. 범위 (Scope)

### In Scope

**소비자 플로우**
- 홈 (근처 인기 상품 목록)
- 상품 상세
- 찜 추가/제거
- 예약하기 (일시 선택 → 확정)
- 선착순 구매
- 내 활동 (찜 목록, 예약 내역, 구매 내역)

**소상공인 플로우**
- 대시보드 (판매 현황 요약)
- 상품 등록 (일반 / 선착순)
- 상품 목록 관리
- 예약 확인/처리

**관리자 플로우**
- 사용자 관리 (목록, 상태 변경)
- 상품 검수 (강제 종료)

### Out of Scope
- 실제 HTML/CSS 구현 (Phase 6)
- 인증/로그인 화면 (Phase 4에서 API와 함께 설계)
- 결제 화면 (후순위)
- 알림 화면 (후순위)

## 4. 사용자 유형별 핵심 플로우 (안)

### 소비자

```
[홈] 인기 상품 목록 (위치 기반)
  └─> [상품 상세]
        ├─> [찜 추가/제거]
        ├─> [예약하기]
        │     └─> [예약 완료]
        └─> [선착순 구매]
              └─> [구매 완료]

[마이페이지]
  ├─> [찜 목록]
  ├─> [예약 내역]
  └─> [구매 내역]
```

### 소상공인

```
[대시보드] 오늘 예약 수 / 판매 수 / 인기점수 요약
  ├─> [상품 등록] (일반 / 선착순)
  ├─> [상품 목록]
  │     └─> [상품 수정 / 종료]
  └─> [예약 관리]
        └─> [예약 확인 / 거절]
```

### 관리자

```
[대시보드] 플랫폼 현황 (전체 사용자 수, 거래 수)
  ├─> [사용자 관리]
  │     └─> [계정 정지 / 강제 탈퇴]
  └─> [상품 검수]
        └─> [강제 종료(FORCE_CLOSED)]
```
